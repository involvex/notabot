NotABot Documentation

Complete guide to using NotABot CLI agent

Quick Start Guide

Get NotABot up and running in minutes with this quick start guide.

1. Installation

npm install -g notabot

2. First Run

notabot

3. Authentication

/login
Note: You'll need to set up Google OAuth credentials first. See the OAuth Setup guide below.

OAuth Setup

Configure Google OAuth authentication for secure access to NotABot.

Step 1: Create Google Cloud Project

  1. Go to Google Cloud Console
  2. Create a new project or select existing
  3. Enable the Gemini API

Step 2: Create OAuth Credentials

  1. Go to "APIs & Services" > "Credentials"
  2. Create OAuth 2.0 Client ID
  3. Add http://localhost:3000/auth/callback to authorized redirect URIs

Step 3: Set Environment Variables

export GOOGLE_CLIENT_ID="your-client-id" export GOOGLE_CLIENT_SECRET="your-client-secret"
Security: Never commit your OAuth credentials to version control. Use environment variables or secure configuration files.

Database System

NotABot uses SQLite for persistent storage of conversations, settings, and user data.

Database Features

  • Conversation history with search capabilities
  • Session tracking and statistics
  • Settings persistence across sessions
  • User profile management
  • File indexing and search
  • Auto mode logging
  • Web server access logs

Database Commands

/db status - Show database statistics /db conversations [10] - Show recent conversations /db search - Search conversation history /db cleanup [30] - Clean up old data (days) /db backup [path] - Create database backup

Database Location

The SQLite database is stored at:

~/.notabot/notabot.db

Web Dashboard

Access NotABot's web interface for remote monitoring and control.

Starting the Web Server

/webserver start

Accessing the Dashboard

Once started, visit: http://localhost:4000

Dashboard Features

  • Real-time session statistics
  • Live conversation history
  • Settings management
  • Remote command execution
  • Auto mode configuration
  • YOLO mode toggle

Web Server Commands

/webserver start - Start web server /webserver stop - Stop web server /webserver status - Check server status

Auto Mode

Configure automated tasks and scheduled operations.

Enabling Auto Mode

/auto enable

Configuring Auto Commands

/auto commands "command1" "command2" "command3"

Setting Auto Triggers

/auto triggers "pattern1" "pattern2"

Auto Mode Commands

/auto enable - Enable auto mode /auto disable - Disable auto mode /auto status - Check auto mode status /auto commands - Set auto commands /auto triggers - Set auto triggers
Tip: Auto mode runs commands in the background at regular intervals. Use with caution and test thoroughly.

Tool System

NotABot includes a comprehensive set of tools for system operations.

Available Tools

@list_directory path=. - List files and directories @read_file path=file - Read file contents @write_file path=file - Write content to file @run_shell_command cmd - Execute shell command @read_all_files path=. - Read all files in directory @cd path=../ - Change directory @yolo_mode action=enable - Enable YOLO mode @code_analysis path=file - Analyze code file

Tool Usage Examples

# List current directory @list_directory path=. # Read a file @read_file path=package.json # Write to a file @write_file path=test.txt content=Hello World # Execute a command @run_shell_command command=ls -la # Change directory @cd path=../src
YOLO Mode: Some tools can be dangerous. Use YOLO mode with extreme caution and only when you understand the implications.

YOLO Mode

YOLO mode allows dangerous operations that are normally restricted.

Enabling YOLO Mode

/yolo
⚠️ Warning: YOLO mode disables safety restrictions. Use only when you understand the risks and have proper backups.

YOLO Mode Features

  • Dangerous file operations
  • System-level commands
  • Bulk file operations
  • Network operations
  • Process management

Disabling YOLO Mode

/yolo

Troubleshooting

Common issues and their solutions.

OAuth Issues

Problem: "App not found" error during login
Solution: Check your OAuth credentials and ensure the redirect URI is correct

Database Issues

Problem: Database initialization fails
Solution: Check file permissions and ensure the .notabot directory is writable

Web Server Issues

Problem: Port 4000 already in use
Solution: Change the port in settings or stop the conflicting service

Getting Help

  • Use /settings to view current configuration
  • Check /stats for session information
  • Visit the GitHub Issues page
